# АИ Блокнот

Готовый Vite/React-проект для голосового AI-блокнота на русском языке. Интерфейс собран в компактный рабочий экран с быстрыми разделами для телефона и ПК.

## Что доработано

- Компактный адаптивный дизайн для телефона и ПК: быстрые вкладки, папки, голос, календарь и настройки без длинной вертикальной ленты.
- Папки и управление вынесены в быстрый раздел: настройки голоса, уведомления, список папок, очистка папки/блокнота.
- Микрофон, ручной ввод команд, статус и быстрые даты собраны в отдельный раздел.
- Календарь открыт по умолчанию и адаптирован под телефон.
- Добавлен PWA manifest и SVG-иконка для установки на главный экран телефона.
- Уведомления синхронизируются в service worker и сохраняются в IndexedDB телефона (`smart_voice_notebook_reminders_db_v1`).
- Клик по уведомлению открывает нужную запись через сообщение service worker или URL `/?openNote=<id>`.
- Расширены папки и словари AI-раскладки: Финансы, Документы, Работа, Дом, Машина, Семья, Здоровье, Учёба, Путешествия, Рецепты, Спорт, Животные, Личное и другие.
- Добавлены голосовые команды для интерфейса: открыть/закрыть календарь, открыть/закрыть настройки, включить/выключить уведомления.
- Исправлена логика второго уведомления: оно теперь берёт дату события, а не дату первого напоминания, и уважает время конкретной записи.
- В workflow GitHub Actions URL dispatcher можно задавать через переменную `REMINDERS_DISPATCH_URL`.

## Локальный запуск

```bash
npm ci
npm run dev -- --host 0.0.0.0
```

На телефоне в той же Wi‑Fi сети откройте `http://IP_КОМПЬЮТЕРА:5173`. Для реальных push-уведомлений после закрытия страницы нужен HTTPS-деплой.

## Проверка сборки

```bash
npm run smoke:voice
npm run build
```

Обе команды должны завершиться без ошибок.

## Уведомления на телефоне

Есть два уровня доставки:

1. Локальная память телефона: приложение отправляет расписание в service worker, service worker пишет напоминания в IndexedDB и восстанавливает таймеры при активации, sync и periodicsync.
2. Серверный Web Push: нужен для более надёжной доставки после полного закрытия браузера. Для него на Vercel задаются `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`, `BLOB_READ_WRITE_TOKEN`, а GitHub Actions вызывает `/api/reminders-dispatch` каждые 5 минут.

На Android лучше проверять через Chrome и HTTPS. Для iPhone уведомления стабильнее после установки PWA на главный экран.

## Vercel

1. Загрузить проект в GitHub.
2. Import Project в Vercel.
3. Framework Preset: Vite.
4. Build Command: `npm run build`.
5. Output Directory: `dist`.
6. Deploy.

`.env.local` не должен попадать в архив и репозиторий. Используйте `.env.example` как шаблон.

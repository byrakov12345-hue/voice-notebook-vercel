# Правила классификации товаров (АИ Блокнот)

Этот файл фиксирует эталонную логику раскладки покупок по спискам.

## Цель
- Любая фраза про покупки должна попадать в корректную категорию.
- Если в одной фразе несколько категорий, запись делится на несколько списков.
- Нельзя смешивать `Еда` и `Инструмент` в одном списке.

## Категории и сигналы

### Еда
- Сигналы: `хлеб`, `батон`, `молоко`, `сыр`, `масло`, `яйца`, `мясо`, `курица`, `рис`, `гречка`, `макароны`, `пельмени`, `сладости`, `чай`, `кофе`.

### Инструмент
- Сигналы: `биты`, `диски`, `болгарка`, `шуруповерт`, `дрель`, `сверло`, `перфоратор`, `пленка`, `саморезы`, `гвозди`, `рулетка`, `ножовка`.

### Аптека
- Сигналы: `аптека`, `лекарство`, `таблетки`, `анальгин`, `ибупрофен`, `парацетамол`, `бинт`, `мазь`, `витамины`.

### Авто
- Сигналы: `для машины`, `авто`, `лобовое`, `шины`, `масло`, `фильтр`, `дворники`, `антифриз`, `аккумулятор`.

### Дом
- Сигналы: `моющее`, `порошок`, `губки`, `посуда`, `бумага`, `для дома`.

### Техника
- Сигналы: `телефон`, `ноутбук`, `планшет`, `кабель`, `зарядка`, `наушники`, `мышка`, `клавиатура`.

## Контекстные подсказки (сильные)
- `для машины`, `для авто` -> `Авто`
- `в аптеку`, `из аптеки` -> `Аптека`
- `для ремонта`, `в мастерскую` -> `Инструмент`
- `к чаю`, `на завтрак`, `на ужин`, `перекус` -> `Еда`

Контекст применяется только как подсказка, если категория не определилась по самому товару.

## Приоритеты и конфликт
1. Сначала категория определяется по самому товару.
2. Если не определилась — по сильной контекстной подсказке.
3. Если не определилось — `Покупки`.
4. Если в фразе смешаны категории — создаются отдельные списки.

## Примеры
- `купить пленку, диски для болгарки, биты, молоко, яйца`  
  -> `Инструмент: пленка, диски, биты` + `Еда: молоко, яйца`

- `купить для машины лобовое стекло и антифриз`  
  -> `Авто`

- `купить в аптеку анальгин и бинт`  
  -> `Аптека`

- `к чаю батон и сыр`  
  -> `Еда`

## Требование к дальнейшим правкам
- Любое изменение классификации делается только с обновлением этого файла.
- Новые сигналы добавлять в категорию, не ломая текущие кейсы пользователя.
